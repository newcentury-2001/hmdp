# hmdp-redis-rate-limit-framework 扩展点与临时封禁策略

本文档介绍该模块的限流扩展点、默认实现以及临时封禁策略的配置与使用方法。

## 模块概览
- 支持 IP 与用户双维度限流，固定窗口与滑动窗口两种算法。
- 提供扩展点以便在限流前后插入自定义逻辑，以及在阻断时执行惩罚策略（如临时封禁）。
- 具备白名单、封禁前置检查，确保在执行 Lua 脚本前即可快速放行或阻断。

## 扩展点

### RateLimitContext
- 作用：在限流流程中传递关键上下文数据，供事件监听与惩罚策略使用。
- 字段：
  - `voucherId`：当前秒杀券 ID
  - `userId`：当前用户 ID
  - `clientIp`：客户端 IP（从请求头中解析）
  - `keys`：本次限流使用的 Redis 键集合
  - `useSliding`：是否使用滑动窗口算法
  - `ipWindowMillis` / `ipMaxAttempts`：IP 限流窗口与最大尝试次数
  - `userWindowMillis` / `userMaxAttempts`：用户限流窗口与最大尝试次数
  - `result`：Lua 执行结果（`BaseCode` 对应的整型编码）

### RateLimitEventListener
- 方法：
  - `onBeforeExecute(ctx)`：Lua 执行前触发，可做审计、打点、灰度参数调整等。
  - `onAllowed(ctx)`：限流通过触发，可做成功监控或链路追踪。
  - `onBlocked(ctx, reason)`：限流阻断触发，可做告警、计数、黑灰名单调整等。
- 默认实现：`NoOpRateLimitEventListener`
  - 提供基础日志与原子计数（`getBeforeExecuteCount()` / `getAllowedCount()` / `getBlockedCount()`）。
  - 在阻断场景输出 `warn` 日志，其余为 `debug`。
  - 你可以在业务工程中声明自己的 `RateLimitEventListener` Bean 覆盖默认实现。

### RateLimitPenaltyPolicy
- 方法：
  - `apply(ctx, reason)`：当发生阻断时执行惩罚策略（如临时封禁、打标签、告警）。
- 默认实现：`NoOpRateLimitPenaltyPolicy`（不做任何处理）。
- 可选实现：`ThresholdPenaltyPolicy`（阈值型临时封禁策略，需开启开关）。

## 临时封禁策略（ThresholdPenaltyPolicy）
- 设计：在一个统计窗口内，累计被阻断次数达到阈值则对 IP 或用户设置封禁标记并指定 TTL。
- 触发时机：当 Lua 返回 `SECKILL_RATE_LIMIT_IP_EXCEEDED` 或 `SECKILL_RATE_LIMIT_USER_EXCEEDED` 时。
- 计数键：
  - IP：`seckill:violation:ip:{voucherId}:{ip}`（`SECKILL_VIOLATION_IP_TAG_KEY`）
  - 用户：`seckill:violation:user:{voucherId}:{userId}`（`SECKILL_VIOLATION_USER_TAG_KEY`）
- 封禁键：
  - IP：`seckill:block:ip:{voucherId}:{ip}`（`SECKILL_BLOCK_IP_TAG_KEY`）
  - 用户：`seckill:block:user:{voucherId}:{userId}`（`SECKILL_BLOCK_USER_TAG_KEY`）
- 自动配置：在 `RateLimitAutoConfiguration` 中基于开关注册。
  - 开启 `rate-limit.enable-penalty=true` → 注入 `ThresholdPenaltyPolicy`
  - 否则默认注入 `NoOpRateLimitPenaltyPolicy`

## 处理器执行流程（RedisRateLimitHandler#execute）
1. 解析客户端 IP（`X-Forwarded-For` 优先，然后 `X-Real-IP`，最后 `remoteAddr`）。
2. 白名单放行：命中 `ipWhitelist` 或 `userWhitelist` 直接返回。
3. 封禁前置检查：命中封禁标记键直接抛出对应错误码。
4. 构建限流键与参数（根据固定窗口或滑动窗口选择键与 Lua）。
5. 构建 `RateLimitContext`，安全触发 `onBeforeExecute`。
6. 执行 Lua，写入结果到 `RateLimitContext`。
7. 结果处理：
   - 允许：触发 `onAllowed`。
   - 阻断：触发 `onBlocked` 并执行惩罚策略 `apply`，随后抛出错误码。

## 配置项（prefix: `rate-limit`）
以下属性位于 `SeckillRateLimitConfigProperties` 中，Spring Boot 自动绑定：

```yaml
rate-limit:
  # 算法选择
  enable-sliding-window: false

  # IP 限流
  ip-window-millis: 5000
  ip-max-attempts: 3

  # 用户 限流
  user-window-millis: 60000
  user-max-attempts: 5

  # 白名单（可选）
  ip-whitelist:
    - 127.0.0.1
  user-whitelist:
    - 1

  # 临时封禁策略（可选）
  enable-penalty: true
  violation-window-seconds: 60
  ip-block-threshold: 5
  user-block-threshold: 5
  ip-block-ttl-seconds: 300
  user-block-ttl-seconds: 300
```

> 说明：属性名采用 Spring Boot 推荐的短横线（kebab-case）风格，对应 Java 字段 `enableSlidingWindow` 等。

## Redis 键参考（部分）
- 限流计数（固定窗口）：
  - IP：`seckill:limit:ip:{voucherId}:{ip}`（`SECKILL_LIMIT_IP_TAG_KEY`）
  - 用户：`seckill:limit:user:{voucherId}:{userId}`（`SECKILL_LIMIT_USER_TAG_KEY`）
- 限流计数（滑动窗口）：
  - IP：`seckill:limit:ip:sw:{voucherId}:{ip}`（`SECKILL_LIMIT_IP_SW_TAG_KEY`）
  - 用户：`seckill:limit:user:sw:{voucherId}:{userId}`（`SECKILL_LIMIT_USER_SW_TAG_KEY`）
- 临时封禁标记：
  - IP：`seckill:block:ip:{voucherId}:{ip}`（`SECKILL_BLOCK_IP_TAG_KEY`）
  - 用户：`seckill:block:user:{voucherId}:{userId}`（`SECKILL_BLOCK_USER_TAG_KEY`）
- 违规计数：
  - IP：`seckill:violation:ip:{voucherId}:{ip}`（`SECKILL_VIOLATION_IP_TAG_KEY`）
  - 用户：`seckill:violation:user:{voucherId}:{userId}`（`SECKILL_VIOLATION_USER_TAG_KEY`）

## 示例：自定义事件监听器
```java
@Component
public class MonitoringRateLimitEventListener implements RateLimitEventListener {
    @Override
    public void onBeforeExecute(RateLimitContext ctx) {
        // 记录即将执行的请求特征
    }
    @Override
    public void onAllowed(RateLimitContext ctx) {
        // 记录通过请求指标
    }
    @Override
    public void onBlocked(RateLimitContext ctx, BaseCode reason) {
        // 记录阻断原因与上下文，可能发送告警
    }
}
```

## 最佳实践
- 区分 IP 与用户维度：IP 阈值通常更严格；用户阈值需谨慎避免误封。
- 选择合理窗口与 TTL：窗口用于统计计数，TTL 决定封禁时长。
- 白名单优先：对可信来源（如内部网关）配置白名单，减少无效限流。
- 控制日志级别：默认监听器 `debug`/`warn` 仅作轻度审计，生产环境可根据需要调整。

## 兼容性说明
- 扩展点均为可插拔实现，默认行为保持向后兼容。
- 关闭 `enable-penalty` 时仅进行常规限流，不产生封禁副作用。

## 启用步骤速览
1. 根据业务需求设置 `rate-limit` 相关参数。
2. 如需临时封禁，开启 `enable-penalty` 并调整阈值与 TTL。
3. 如需更丰富的审计/告警，提供自定义 `RateLimitEventListener` Bean 覆盖默认实现。
